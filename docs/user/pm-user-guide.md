# PM User Guide

Daily workflow, approval queue, weekly reports, and risk management.

## Daily Workflow

### Morning

1. `ai-pm daily --format text` — Review priorities, blockers, and risks.
2. `ai-pm approval list --json` — Check pending approvals.
3. `ai-pm project scan --json` — Verify readiness score.

### During the Day

- Approve or reject agent actions: `ai-pm approval decide <id> approve`
- Track risks: `ai-pm risk list --json`
- Monitor memory: `ai-pm memory summary --json`

### End of Day

- `ai-pm daily --format markdown --output drafts/` — Save daily draft.
- Review audit trail: `ai-pm audit list --json`

## Approval Queue

The approval queue holds all external mutations proposed by agents:

| Status | Meaning |
|--------|---------|
| `pending` | Awaiting PM decision |
| `approved` | Decision made, execution pending |
| `rejected` | PM declined |
| `revision_requested` | PM asked for changes |
| `expired` | Deadline or TTL exceeded |
| `executing` | Being executed |
| `executed` | Done |
| `execution_failed` | Execution error |

### Approve/Reject from CLI

```bash
ai-pm approval decide <id> approve
ai-pm approval decide <id> reject --reason "Not needed right now"
ai-pm approval decide <id> revision --notes "Add more detail"
```

### Approval Policy

All external mutations (email, Jira, GitHub, Notion, Confluence) require PM approval before execution. Local operations (memory, audit) do not require approval.

## Weekly Reports

```bash
ai-pm weekly --start 2026-06-16 --end 2026-06-21 --format markdown --output drafts/
ai-pm weekly --json
```

Weekly reports include: accomplishments, next week focus, risks, decisions, milestones, and confidence score.

## Risk Management

```bash
ai-pm risk list --json
```

Risks are tracked in `risks/risk-register.md`. Each risk includes: statement, category, probability, impact, owner, mitigation, and status.

## Memory and Artifacts

```bash
ai-pm memory summary --json
ai-pm memory tasks --json
ai-pm memory artifacts --json
```

Memory tracks tasks, artifacts, and their relationships. Artifacts are generated by workflows and persisted locally.

## Audit Trail

```bash
ai-pm audit list --json
```

Every workflow run, approval decision, and external mutation is recorded in the audit trail at `.ai-pm/audit/`.
