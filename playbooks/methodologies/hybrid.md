# Hybrid Methodology Playbook

## Use When

Use Hybrid when high-level governance is phase-based but delivery inside phases uses iterative or agile practices.

## Required Artifacts

- phase roadmap
- milestone plan
- agile backlog
- sprint or flow board
- integration plan
- release plan
- phase sign-off records

## Governance Pattern

- Use Waterfall-style gates for phase approval, contract milestones, and client sign-off.
- Use Scrum or Kanban inside delivery phases for iterative execution.
- Keep traceability from phase scope to backlog items.

## Control Metrics

- phase milestone variance
- sprint or flow health
- dependency risk
- integration readiness
- requirements volatility
- release confidence

## Agent Rules

- Never let sprint-level flexibility hide phase-level scope changes.
- Keep client commitments tied to phase milestones.
- Use agile metrics for team execution and phase-gate metrics for governance.
- Escalate cross-team dependencies early.

## Reporting Focus

- phase-level health
- iteration progress
- integration risks
- dependency status
- approval needs
