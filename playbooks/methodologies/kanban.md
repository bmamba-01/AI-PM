# Kanban Methodology Playbook

## Use When

Use Kanban when work flows continuously and the team optimizes throughput, WIP, and cycle time instead of sprint commitments.

## Required Artifacts

- workflow board
- WIP limits
- blocked-item policy
- service level expectation
- cumulative flow report

## Ceremonies

- flow review
- replenishment
- delivery planning
- service delivery review
- operations review

## Control Metrics

- WIP count
- cycle time
- lead time
- throughput
- aging work items
- blocked duration
- flow efficiency

## Agent Rules

- Flag WIP limit breaches.
- Escalate aging work items before they become delivery surprises.
- Treat blocked duration as a risk signal.
- Recommend policy changes only with observed flow evidence.

## Reporting Focus

- flow health
- bottlenecks
- blocked work
- delivery predictability
- throughput trends
