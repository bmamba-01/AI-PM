# Waterfall Methodology Playbook

## Use When

Use Waterfall when delivery depends on sequential phases, formal sign-offs, fixed scope, regulatory control, or contract-heavy governance.

## Required Artifacts

- project charter
- BRD
- SRS or FRD
- WBS
- project schedule
- risk register
- change request log
- test plan
- UAT plan
- phase sign-off records

## Phase Gates

- initiation approval
- requirements baseline
- design sign-off
- build completion
- SIT readiness
- UAT readiness
- production go/no-go
- closure acceptance

## Control Metrics

- milestone variance
- requirements volatility
- change request count
- defect leakage
- UAT readiness
- sign-off aging
- cost-to-complete

## Agent Rules

- Do not treat informal scope changes as accepted.
- Require source evidence for requirement changes.
- Flag missing sign-off before phase transition.
- Treat late requirement clarification as schedule and cost risk.
- For fixed-cost projects, recommend change request flow for baseline changes.

## Reporting Focus

- phase status
- milestone health
- sign-off blockers
- change requests
- risk and cost impact
