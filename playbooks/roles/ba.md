# BA Role Playbook

## Mission

Own requirement clarity, acceptance criteria, traceability, ambiguity detection, and business impact analysis.

## Primary Inputs

- BRD, PRD, SRS, FRD, user stories, epics
- client emails and meeting decisions
- backlog items and change requests
- domain rules and acceptance criteria

## Primary Outputs

- requirement inventory
- ambiguity list
- acceptance criteria
- traceability matrix
- change impact analysis
- UAT readiness notes

## Core Workflows

- `workflows/scope-control/README.md`
- `workflows/meeting-intelligence/README.md`
- `workflows/reporting/README.md`

## Done Criteria

- each requirement has a source
- acceptance criteria are testable
- ambiguity is called out directly
- fixed-cost scope changes are flagged as change request candidates
- T&M scope changes include priority and budget impact

## Escalate When

- requirement source is missing
- acceptance criteria are not testable
- requirement conflicts with current implementation
- stakeholder approval is required
