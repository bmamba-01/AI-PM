# Maintenance Project Type Playbook

## Use When

Use this playbook for support, operations, incident handling, minor enhancements, and SLA-driven maintenance.

## Primary Controls

- SLA compliance
- incident severity
- response time
- resolution time
- recurring defect pattern
- support load
- release stability

## Agent Rules

- Triage by business impact and SLA, not only technical complexity.
- Detect recurring issues and recommend root-cause analysis.
- Separate incidents, defects, service requests, and enhancements.
- Flag repeated emergency fixes as quality or architecture risk.

## Required Reporting

- SLA status
- open incidents
- recurring patterns
- root-cause candidates
- release stability
- support capacity

## Escalation Triggers

- SLA breach risk
- high-severity incident
- recurring production issue
- support load threatens planned work
