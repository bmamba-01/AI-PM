# Fixed Cost Project Type Playbook

## Use When

Use this playbook for projects with fixed scope, fixed price, milestone acceptance, or margin constraints.

## Primary Controls

- scope baseline
- change request log
- milestone acceptance
- cost-to-complete
- margin risk
- requirements volatility
- acceptance criteria
- sign-off status

## Agent Rules

- Treat scope ambiguity as commercial risk.
- Flag any work outside baseline as a change request candidate.
- Require impact analysis for changes to scope, timeline, or acceptance criteria.
- Do not silently absorb client changes into delivery work.
- Track sign-off aging and acceptance blockers.

## Required Reporting

- baseline scope status
- approved changes
- pending change requests
- milestone status
- cost-to-complete risk
- client decisions needed

## Escalation Triggers

- scope change lacks approved change request
- milestone acceptance is blocked
- requirements change after baseline
- margin risk increases
- client expectation differs from signed scope
