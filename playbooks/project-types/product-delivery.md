# Product Delivery Project Type Playbook

## Use When

Use this playbook for product teams optimizing roadmap outcomes, customer value, discovery, adoption, and iterative delivery.

## Primary Controls

- roadmap alignment
- outcome metrics
- discovery insights
- customer feedback
- feature adoption
- release readiness
- backlog priority

## Agent Rules

- Connect features to outcomes and measurable value.
- Distinguish stakeholder requests from validated user needs.
- Surface roadmap trade-offs clearly.
- Use feedback and adoption data when available.
- Avoid treating output shipped as outcome achieved.

## Required Reporting

- roadmap progress
- outcome confidence
- discovery findings
- customer impact
- release risk
- next prioritization decisions

## Escalation Triggers

- roadmap commitment conflicts with discovery evidence
- feature lacks success metric
- adoption risk is high
- release scope threatens outcome focus
